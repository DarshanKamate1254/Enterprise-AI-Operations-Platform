Platform Reference & Reference Architecture

An Enterprise-Grade Multi-Agent AI System Orchestrated by LangGraph with Distributed State checkpointers, MCP tools execution, and Microservice decupled layouts.

The **AI_OOPS AI Operations Platform** is a flagship multi-agent system built for advanced corporate governance. It translates messy, unstructured operational requests into organized steps, fetches policies using retrieval-augmented intelligence, runs relational SQL queries, coordinates third-party APIs, and scans all outputs for data safety and compliance.

Rather than a monolithic setup, the platform is designed around decoupled, independent services communicating via standard APIs. The multi-agent orchestrator uses LangGraph to coordinate cycles and reflection loops.

Architecture & Execution Flow

The Runtime Execution Flow

When a query is received, the request flows sequentially through the system:

1
Security & Intent Classification
The request is scanned for prompt injection attacks and evaluated against table-level role boundaries.
Router Agent Security Guard
2
Operational Decomposition
The request is mapped to a sequence of execution steps (SQL query, RAG retrieval, or REST API calls).
Planner Agent
3
Asynchronous Parallel Execution
The platform invokes the required specialized execution agents in parallel.
SQL Agent Retrieval Agent API Agent
4
Quality Assurance & Correctness Audits
Checks if the output answers the query correctly. If database errors or empty sets are returned, it loops back to the Planner.
Reflection Agent
5
PII Scan & Compliance Validation
Exposed outputs are scanned, sensitive data (emails, credit cards, password hashes) is redacted, and safety is verified.
Safety Agent
6
Final Report Compilation
The approved and sanitized context is compiled into a formatted Markdown report.
Report Agent

Core Techniques & Rationale

Building a production-ready AI system requires specific design patterns to handle the unpredictability of LLMs. Here is a breakdown of the techniques used in the project and the rationale behind them:

Technique Implementation Details Why (Architectural Rationale)
Cyclic Workflows (LangGraph) Uses LangGraph's StateGraph to manage cycles, conditional edges, and feedback loops. Unlike linear pipelines, operations require loops (e.g., self-correction). LangGraph maintains state across retries cleanly.
Model Context Protocol (MCP) Implements an HTTP server hosting platform tools (SQL query, retriever, filesystem, math, memory, rest_api). Separates agent intelligence from tool execution. Agents execute tasks via the MCP server, simplifying tool management.
AST Math Parser Math tasks are processed via Python's ast module instead of standard eval(). Protects the server from arbitrary code execution attacks, permitting only secure operations.
Hybrid Search & Reranking Similarity search in Qdrant combined with a local Flashrank reranker. Vector search can retrieve irrelevant chunks. Reranking sorts results by semantic relevance, improving context quality.
PII Masking & Regex Scans Applies pre-response regex filters to redact emails, 16-digit credit cards, and bcrypt password hashes. Prevents the leakage of sensitive data (PII) to end-users or model logs.
Redis State Checkpointer LangGraph checkpointers are backed by a distributed Redis database. Ensures session persistence and chat history restoration across distributed gateway nodes.

The 8 Specialized Agents

The system distributes responsibilities across 8 specialized agent roles, each defined by system prompts and structured output requirements:

1. Intent Router Agent

Determines the execution path for incoming queries. Uses Pydantic schemas (RouteDecision) to select targets: sql, rag, api, general, or unsafe. It intercepts SQL injections and unauthorized table access.

2. Task Planner Agent

Generates a sequential list of steps needed to answer complex queries. If reflection fails, the planner receives critique feedback and adjusts the plan. Includes logic to summarize history when thread context grows, saving token costs.

3. Retrieval Agent (RAG)

Performs similarity queries in Qdrant collection documents. By routing requests through the MCP retriever tool, it retrieves policy chunks and reranks them via Flashrank.

4. SQL Database Agent

Translates natural language questions into read-only SQL queries. It inspects schema types and enforces a whitelist policy (only allowing SELECT commands on public tables).

5. API Client Agent

Constructs and triggers outgoing REST HTTP queries. Calls the MCP rest_api tool to fetch external payloads.

6. Reflection Agent (QA)

Acts as a quality control layer. It checks database outcomes and retrieval contexts. If it detects empty tables or irrelevant text chunks, it triggers a retry loop back to the Planner.

7. Safety Compliance Agent

Scans response buffers before final compilation. It redacts PII data (credit cards, hashes, emails) to ensure compliance.

8. Report Compilation Agent

Consolidates all intermediate states and query outcomes into a structured Markdown document using tables and bullet points.

Unified Tooling Ecosystem

The platform isolates all actions inside a dedicated tools layer. The MCP Server hosts these tools, allowing agents to execute operations via standard API models.

Each tool also supports a **local fallback mechanism**. If the MCP server is unreachable, the agents execute the tool locally using backup code blocks, preventing downtime. 

# Example MCP call structure
POST http://localhost:8080/mcp
Headers: { "Authorization": "Bearer token_here" }
Payload:
{
  "tool": "sql_query",
  "arguments": {
    "query": "SELECT name FROM departments WHERE cost_center = 'CC-100'"
  }
}

Containerization, DevOps & CI/CD

The platform is designed to run in containerized environments. It includes Docker configuration files, Kubernetes manifests, and automation scripts:

Docker Compose

Orchestrates Python services, React frontend nodes, and databases (Postgres, Redis, Qdrant, Prometheus, Grafana). Includes health checks to ensure dependencies are running before dependent services start.

Kubernetes manifests

Provides configurations for production-scale deployments:

  • StatefulSets: Manages PostgreSQL, Redis, and Qdrant database pods, ensuring persistent storage volumes (PVCs) remain attached.
  • HPAs (Horizontal Pod Autoscalers): Automatically scales the API Gateway deployment based on CPU load.
  • Ingress Rules: Route external requests to the correct services (frontend dashboard to port 80, API calls to the Gateway API on port 8000).

CI/CD Pipeline (GitHub Actions)

Automates code verification:

  1. Installs dependencies and runs Python unit and integration tests (pytest).
  2. Runs TypeScript compilation checks (npm run build) for the React frontend.
  3. Validates Docker Compose configuration files.
